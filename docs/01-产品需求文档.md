# 📋 产品需求文档 (PRD)

## 项目概述

**项目名称**：基于 WEB 的房屋信息展示平台  
**项目类型**：Web 应用  
**版本**：v1.0  
**更新日期**：2026-02-25  

---

## 一、项目背景

### 1.1 业务背景

为了做好用户连接，将用户引导到自由信息平台，便于后期成交。

### 1.2 目标用户

- **购房者/租房者**：查找房源信息，联系销售顾问
- **销售顾问**：接收客户咨询，跟进潜在客户
- **平台管理员**：管理房源信息，管理销售人员

### 1.3 业务目标

1. 展示房源信息，吸引潜在客户
2. 收集用户联系方式，建立用户连接
3. 引导用户到自由信息平台，促进后期成交

---

## 二、功能需求

### 2.1 用户系统

#### 2.1.1 游客模式
- 浏览房源列表
- 查看房源详情
- 限制：无法查看完整联系方式

#### 2.1.2 用户注册/登录
- 注册方式：手机号 + 验证码 / 邮箱 + 密码
- 登录方式：账号密码 / 手机验证码
- 第三方登录：微信（可选）

#### 2.1.3 用户信息
- 基本信息：姓名、手机号、邮箱
- 看房记录：浏览过的房源
- 咨询记录：联系过的销售

### 2.2 房源展示

#### 2.2.1 房源列表
- 筛选条件：区域、价格、户型、面积
- 排序：最新、价格升序/降序
- 分页展示

#### 2.2.2 房源详情
- 基本信息：标题、价格、面积、户型、区域
- 详细描述：房屋介绍、周边配套
- 图片展示：户型图、实景图（至少 5 张）
- 位置信息：地图定位、交通配套
- 销售顾问：头像、姓名、电话、微信

### 2.3 销售对接

#### 2.3.1 联系方式展示
- 电话号码（点击拨打）
- 微信二维码（长按识别）
- 在线咨询（跳转自由信息平台）

#### 2.3.2 咨询表单
- 用户姓名
- 联系电话
- 咨询内容
- 提交后销售主动联系

### 2.4 后台管理（MVP 可简化）

#### 2.4.1 房源管理
- 添加/编辑/删除房源
- 房源上下架
- 图片上传

#### 2.4.2 销售管理
- 添加/编辑销售顾问
- 分配房源给销售

#### 2.4.3 用户管理
- 查看注册用户
- 查看咨询记录

---

## 三、非功能需求

### 3.1 性能要求
- 页面加载时间 < 3 秒
- 支持并发用户 1000+

### 3.2 安全要求
- 用户密码加密存储
- 手机号脱敏展示
- 防爬虫机制

### 3.3 兼容性
- 支持 PC 端和移动端
- 主流浏览器兼容（Chrome、Safari、微信内置浏览器）

---

## 四、技术选型建议

### 4.1 前端
- 框架：Vue 3 / React
- UI 库：Element Plus / Ant Design
- 构建工具：Vite

### 4.2 后端
- 语言：Node.js / Python
- 框架：Express / FastAPI
- 数据库：MySQL / PostgreSQL

### 4.3 部署
- 代码托管：GitHub
- 部署平台：Vercel / Railway / 阿里云
- CI/CD：GitHub Actions
