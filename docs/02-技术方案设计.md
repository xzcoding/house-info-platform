### 2.2 表结构

#### users - 用户表
```sql
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  phone VARCHAR(20) UNIQUE NOT NULL,
  name VARCHAR(50),
  password_hash VARCHAR(255),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_phone (phone)
);
```

#### houses - 房源表
```sql
CREATE TABLE houses (
  id INT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(200) NOT NULL,
  price DECIMAL(10,2) NOT NULL,
  area DECIMAL(8,2) NOT NULL,
  rooms INT NOT NULL,
  hall INT NOT NULL,
  location VARCHAR(200),
  address VARCHAR(500),
  description TEXT,
  agent_id INT,
  status ENUM('on_sale', 'sold', 'off') DEFAULT 'on_sale',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_status (status),
  INDEX idx_agent (agent_id)
);
```

#### house_images - 房源图片表
```sql
CREATE TABLE house_images (
  id INT PRIMARY KEY AUTO_INCREMENT,
  house_id INT NOT NULL,
  image_url VARCHAR(500) NOT NULL,
  image_type ENUM('cover', 'room', 'kitchen', 'bathroom', 'other'),
  sort_order INT DEFAULT 0,
  FOREIGN KEY (house_id) REFERENCES houses(id) ON DELETE CASCADE
);
```

#### agents - 销售顾问表
```sql
CREATE TABLE agents (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(50) NOT NULL,
  phone VARCHAR(20) NOT NULL,
  wechat VARCHAR(100),
  avatar VARCHAR(500),
  status ENUM('active', 'inactive') DEFAULT 'active',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### consultations - 咨询记录表
```sql
CREATE TABLE consultations (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT,
  house_id INT NOT NULL,
  agent_id INT NOT NULL,
  user_name VARCHAR(50),
  user_phone VARCHAR(20),
  message TEXT,
  status ENUM('new', 'contacted', 'completed') DEFAULT 'new',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (house_id) REFERENCES houses(id),
  FOREIGN KEY (agent_id) REFERENCES agents(id)
);
```

## 三、API 设计

### 3.1 用户相关

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/auth/send-code` | POST | 发送验证码 |
| `/api/auth/login` | POST | 登录 |
| `/api/auth/register` | POST | 注册 |
| `/api/user/profile` | GET | 获取用户信息 |
| `/api/user/profile` | PUT | 更新用户信息 |

### 3.2 房源相关

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/houses` | GET | 房源列表 |
| `/api/houses/:id` | GET | 房源详情 |
| `/api/houses` | POST | 创建房源（管理员） |
| `/api/houses/:id` | PUT | 更新房源（管理员） |
| `/api/houses/:id` | DELETE | 删除房源（管理员） |

### 3.3 咨询相关

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/consultations` | POST | 提交咨询 |
| `/api/consultations` | GET | 咨询列表（管理员） |
| `/api/consultations/:id/status` | PUT | 更新状态（管理员） |

### 3.4 销售顾问

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/agents` | GET | 销售列表 |
| `/api/agents/:id` | GET | 销售详情 |

### 6.2 GitHub Actions

```yaml
# 前端部署
name: Deploy Frontend
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install && npm run build
      - uses: amondnet/vercel-action@v20
```
